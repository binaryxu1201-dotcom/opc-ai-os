## 1 文档目的

本文将已评审的 V1-A 产品、架构、数据、API、UI/UX 和测试发布基线转换为可执行的工程计划、任务依赖、验收标准、责任边界和阶段门。

本文只服务于 OPC AI OS V1-A 工程落地，不引入其他项目的代码、目录或技术约束。V1-A 仍采用 TypeScript Monorepo、React Web PWA、TypeScript 模块化 REST 服务、PostgreSQL、Redis 和独立 Worker。

## 2 实施范围冻结

### 2.1 V1-A 必须交付

1. 账户注册、登录、会话刷新/轮换/撤销、重认证。
2. 唯一工作空间、基础画像和独立数据授权。
3. 项目、最多三级个人任务、项目/任务状态机、乐观锁和幂等。
4. CRM-lite：客户、阶段流转、阶段历史。
5. AI COO：项目任务拆解、每日三件事、澄清、建议编辑/确认/驳回、失败降级。
6. 用户数据导出、一次性下载令牌、注销宽限、撤销注销、到期匿名化/删除策略。
7. 审计 append-only、AI traceId、结构化日志、指标和基础运营聚合。
8. React PWA 的移动/桌面核心页面、错误状态、只读状态和无障碍要求。
9. 独立 Worker：导出、AI 重试、注销到期处理，最多重试 3 次并进入失败队列。
10. CI、单元/集成/API 契约/E2E/安全/性能/AI 质量/备份恢复验证。

### 2.2 明确不做

V1-A 不实现也不提前建设：多成员协作、任务协作市场、需求发布、候选匹配、站内沟通、支付/结算、评价、AI CLO、长期记忆、RAG/向量库、通用文件服务、企业主体、多租户成员关系、内容社区实验、站内信、浏览器/邮件/短信推送、公开 API、WebSocket、完整审计查询界面。

### 2.3 工程决策冻结项

| 决策 | V1-A 结论 | 依据 |
|---|---|---|
| 应用形态 | 模块化单体 + 独立 Worker | 03 §2、§4 |
| 前端 | React Web PWA，移动优先 | 03 §1.2、06 §2 |
| 后端 | TypeScript 模块化 REST 服务 | 03 §4.1、05 §1.3 |
| 数据库 | PostgreSQL 16+ 兼容实现；本机已安装 PostgreSQL 17.2 | 04 §2.1；本机检查记录见 §10 |
| 缓存/队列 | Redis 8.8.0（旧版 3.2.100 已升级）；权威业务事实在 PostgreSQL | 03 §4.1、04 §2.3；本机检查记录见 §11 |
| AI 流式 | SSE；不使用 WebSocket | 05 §1.3 |
| 会话 | 短时 Access Token + HttpOnly Secure SameSite Refresh Cookie | 03 §5、05 §1 |
| 隔离 | 服务层强制 `actor_id + workspace_id`，复合外键辅助；不依赖客户端过滤 | 03 §6、04 §1.3 |
| AI 写入 | AI 只产生建议；用户确认后由确定性业务服务事务写入 | 03 §7、04 §1.3 |

## 3 交付策略

### 3.1 原则

1. **先建立可运行骨架，再做业务垂直切片。**
2. **先完成确定性手动经营路径，再接入 AI。**
3. **安全、授权、审计和测试与功能同步交付，不作为收尾工作。**
4. **每个迭代都产出可运行、可测试、可回滚的增量。**
5. **所有范围变更必须更新本计划、影响的上游文档和测试矩阵。**

### 3.2 第一条最小垂直切片

第一条必须打通的工程链路：

```text
注册 → 登录 → 创建工作空间 → 完成画像 → 创建项目 → 创建三级任务 → 查询/变更任务
```

该切片必须同时包含：真实 PostgreSQL 迁移、认证上下文、工作空间隔离、状态机、审计、API 契约、单元测试、集成测试、桌面 E2E 和移动 E2E。没有 AI 也必须完整通过。

## 6 关键依赖关系

```text
工程骨架/配置
  → 数据库迁移/Redis/日志
  → 认证/工作空间/授权
  → 项目/任务/CRM 手动闭环
  → AI 受控链路
  → 导出/注销/Worker
  → PWA 全量体验
  → 全量测试与发布演练
```

允许并行的工作：

- E01-T02~T06 可在同一迁移规范下并行设计，落地时按依赖顺序合并。
- E03 手动业务和 E06 页面框架可以并行，但页面不得绕过 API 契约自行定义业务状态。
- E07-T01 的测试模板可与各模块开发同步建立。
- E04 AI 适配器可以先使用 Mock，不能等待真实供应商才开始契约和降级测试。
