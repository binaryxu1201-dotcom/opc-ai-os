## 1 文档目的与范围

### 1.1 目的

本文将 SRS 中 V1-A（0–6 周）范围细化为可设计、开发、测试和运营验证的产品需求。本文定义用户可见行为、业务规则、页面信息、用户故事、验收条件、埋点及发布准入；不定义技术架构、接口协议、数据表结构、视觉设计、定价或法务结论。

### 1.2 产品目标

验证“AI 能否帮助个人创业者形成持续经营习惯”。用户完成注册、画像和项目创建后，AI COO 协助将项目目标拆解为可确认的任务；用户在工作台查看每日最重要的三件事，管理任务、项目与客户，从而建立可持续的个人经营节奏。

### 1.3 目标用户与账户边界

| 项目 | V1-A 规则 |
|---|---|
| 目标用户 | 1–2 人的初创 OPC，具备核心技能但缺经营管理能力 |
| 登录主体 | 个人登录账户，手机或邮箱注册 |
| 工作空间 | 一个账户仅可创建一个 OPC 工作空间；业务数据归属于该工作空间 |
| 成员模型 | 单用户。不存在团队成员、邀请、组织架构或成员角色 |
| 实名 | 当前 V1-A 基线不启用；不作为注册、创建项目或使用 AI COO 的前置条件。仅保留供应商适配候选，启用前须修订产品/合规/技术文档。 |
| 任务负责人 | 所有 V1-A 任务的负责人固定为当前工作空间所有者；该字段仅为未来扩展保留，不展示成员选择器 |

### 1.4 V1-A 范围

1. 注册、登录、基础画像、工作空间创建和数字身份聚合页。
2. 初版 OPC 阶段标签。
3. 项目、个人任务和子任务管理；项目仪表盘。
4. AI COO：项目任务拆解、每日三件事、建议确认与任务状态联动。
5. CRM-lite：客户录入、阶段管理。
6. 独立数据授权、授权留痕、基础数据导出、账户注销申请。
7. 核心操作审计、AI 调用追踪、基础运营后台和核心运营指标。
8. AI 调用配额作为 V1-A 运营能力实现；内容发布与轻量审核实验在本基线中**明确关闭**，仅保留为候选 backlog，不进入 03–07 的架构、数据、API、UI 或发布范围。

### 1.5 明确非目标

V1-A 不包含：任务协作市场、需求发布、候选匹配、站内沟通、合作协议、结算记录、真实支付、评价、AI CLO、长期记忆、RAG、通用文件上传、站内信和任何浏览器/邮件/短信推送、企业实名、多成员协作、完整会员体系、资源市场与服务市场。

### 1.6 成功与阶段门

V1-A 阶段门的判定以 SRS §2.6.4、§6.7 为准。观察前提为不少于 100 名完成“注册+画像+创建项目”的有效激活用户，并连续观察至少 2 周。必须达标：

| 指标 | 目标 | 产品采集依据 |
|---|---:|---|
| 注册至首次创建项目转化率 | >=50% | `project_created` 在注册后 7 天内发生 |
| AI 任务建议确认率 | >=30% | `ai_suggestion_confirmed` / 已展示建议数 |
| 激活用户 7 日留存 | >=35% | 激活后第 7–13 天至少一次有效登录 |
| 每日三件事页面周活跃率 | >=40% | 激活用户每周至少一次 `daily_top3_viewed` |
| AI 质量测试集通过率 | >=90% | §11 发布前测试结果 |
| 重大安全事件、严重合规问题 | 0 | 安全与运营事件台账 |

## 2 产品原则

1. **AI 提供建议，人做最终决定。** AI 输出本身不改变项目、任务、客户或授权状态；所有写入均须用户明确确认并经过业务规则校验。
2. **确定性操作可直接完成。** 用户可通过普通 UI 手动创建、编辑、完成或取消业务对象，无须经过 AI。
3. **单人优先。** 页面、文案和数据规则围绕个人工作空间设计，不模拟企业协作。
4. **最小数据与可退出。** 仅采集功能必要数据；授权逐项独立；用户可导出核心数据和发起注销。
5. **AI 可解释、可降级、可追溯。** 每次 AI 输出关联 traceId、模型版本、输入摘要和用户处理结果；AI 不可用时，项目和任务仍可手动管理。

## 5 核心业务规则与状态

### 5.1 项目状态

`草稿 -> 进行中 -> 已完成 -> 已归档`；草稿可取消，进行中可暂停或取消。V1-A 支持草稿、进行中、已完成、已暂停、已取消；“已归档”仅作为后续兼容状态，不提供独立归档操作。

项目进入“进行中”的条件为至少存在一项已确认或进行中的任务。所有非取消任务完成后，系统提示用户将项目标记为已完成；系统不得在未确认时自动关闭项目。

### 5.2 个人任务状态

`草稿 -> 待确认 -> 已确认 -> 进行中 -> 已完成 -> 已关闭`。草稿可取消；待确认可编辑、确认或驳回；已确认可开始执行；进行中可完成或标记延期；已完成可关闭；驳回后允许编辑并重新提交确认。

“是否延期”“是否逾期”“用户是否拒绝”均为附加标记，不替代主状态。V1-A 没有待验收、争议中或协作交付状态。

### 5.3 AI 建议状态

`已接收 -> 解析中 -> 已生成建议 -> 等待用户确认 -> 已确认/已拒绝`。确认后才创建或更新相应业务数据；执行失败时保留建议和失败原因，提示用户手动处理。AI 超时、无效结构化输出或服务故障不阻断手工创建项目和任务。

### 5.4 数据授权规则

| 授权项 | V1-A 默认 | 拒绝/撤回后的影响 |
|---|---|---|
| 基础服务必要处理 | 必需 | 无法提供注册、登录、项目和任务基础服务 |
| AI COO 读取项目与任务数据 | 功能所需 | AI COO 不读取业务数据，相关建议不可用；业务数据仍可手动管理 |
| 平台知识与模型改进 | 默认关闭，单独授权 | 停止新的数据用于此目的 |
| 个性化推荐 | 可选 | 停止基于行为的个性化推荐 |
| 营销通知 | 默认关闭 | 不发送营销通知 |

V1-A 不提供长期个人记忆和共享记忆。AI COO 可读取经过授权的项目、任务和客户结构化业务数据，并可使用单次会话上下文；两者均不等同于长期语义记忆或 RAG。

## 7 页面状态与异常处理

| 场景 | 产品行为 |
|---|---|
| 首次进入工作台且无项目 | 说明尚未建立经营计划，提供“创建项目”入口；创建首个项目后再添加任务 |
| 项目无任务 | 显示 AI 拆解和手动新增任务入口，不虚构进度 |
| AI 拆解超时（超过 15 秒） | 提示正在处理，可稍后查看；后台最多重试一次；同时提供手动拆解入口 |
| AI 返回无效结构化结果 | 自动重试一次；仍失败则以明确标记的自然语言建议展示，用户可手动整理为任务 |
| AI 服务不可用 | 显示服务暂不可用；项目、任务、客户手动操作继续可用 |
| 授权被关闭 | 清楚说明受影响能力；不删除业务数据；不再向 AI 提供对应业务数据 |
| 导出处理中或失败 | 显示处理中状态；失败时说明可重试，不暴露内部堆栈 |
| 注销宽限期 | 禁止创建新项目、任务和客户；保留历史查看与撤销注销入口 |
| 越权访问 | 返回无权访问提示，不显示资源存在性或敏感字段 |

## 8 埋点与指标字典

### 8.1 公共字段

所有事件包含：`event_name`、`event_time`、`user_id`（脱敏存储）、`workspace_id`、`session_id`、`trace_id`（涉及 AI 时）、`client_type`、`app_version`。不得将密码、令牌、原始身份证件信息或客户敏感内容写入埋点。

### 8.2 事件清单

| 事件 | 触发时机 | 关键属性 | 用途 |
|---|---|---|---|
| `registered` | 注册成功 | register_method | 注册漏斗 |
| `profile_completed` | 画像提交成功 | stage_label | 激活判定 |
| `workspace_created` | 工作空间创建成功 | — | 激活判定 |
| `project_created` | 项目创建成功 | source=manual/ai_confirmed | 首次项目转化 |
| `ai_task_generation_requested` | 请求 AI 拆解 | project_id | AI 性能与成本 |
| `ai_task_generation_completed` | AI 返回建议 | duration_ms, result_type | 结构化成功率 |
| `ai_suggestion_viewed` | 用户看见建议 | suggestion_count | 确认率分母 |
| `ai_suggestion_confirmed` | 用户确认建议 | edited_count | 确认率分子 |
| `ai_suggestion_rejected` | 用户驳回建议 | reason_optional | 质量改进 |
| `task_status_changed` | 任务状态变更 | before_status, after_status, source | 任务完成率 |

| `consent_changed` | 授权改变 | consent_type, before, after | 授权留痕 |

| `deactivation_requested` | 发起注销 | — | 账户生命周期 |
| `ai_quota_exceeded` | 达到配额 | capability_type | 成本控制 |

### 8.3 口径约束

“激活用户”固定定义为完成注册、基础画像和首次创建项目的用户。测试账户、运营账户和内部压测流量必须在所有阶段门指标中排除。AI 建议确认包含“编辑后确认”，不包含仅查看、忽略或驳回。

## 11 测试与发布验收

### 11.1 核心验收场景

1. 新用户在 3 分钟内完成注册、画像和唯一工作空间创建。
2. 用户手动创建项目与三级任务，所有状态转换符合 §5。
3. AI 在合格输入下生成结构化任务；用户编辑后确认才写入业务数据。
4. AI 故障时，用户可继续手动管理项目、任务和客户。
5. 工作台正确展示每日三件事；无任务时显示空状态，不触发站外推送。
6. 用户只能读取和修改自己的工作空间数据；篡改资源 ID 不得获得其他用户数据。
7. 授权可逐项查询、变更和撤回；每次变更形成审计记录。
8. 用户可导出项目、任务、客户 CSV；注销流程满足 30 天宽限与业务冻结规则。
9. 每次 AI 调用均可关联 traceId、模型版本、输入摘要、输出摘要和用户处理结果。

### 11.2 发布准入

| 条件 | 要求 |
|---|---|
| P0 缺陷 | 全部修复 |
| 核心 E2E | 通过率 100% |
| AI 质量测试集 | >=100 个任务拆解样本，整体通过率 >=90% |
| 权限与越权 | 无高危问题，越权访问成功率为 0 |
| 性能 | §10 指标达标 |
| 安全扫描 | 无高危漏洞 |
| 合规基线 | 数据授权、导出、注销、审计和 AI 数据边界通过法务/安全确认 |

## 12 V1-B 衔接与冻结项

V1-B 仅在 V1-A 阶段门所有必须达标项通过、观察数据完成采集并完成评审后启动。本 PRD 不提前交付 V1-B 的协作市场、实名认证强制、站内通知中心、文件上传、长期记忆、RAG、AI CLO、合作/结算/评价等需求。

为避免返工，V1-A 仅保留以下兼容信息：任务负责人字段固定当前空间所有者；AI 输出与审计均带 traceId；项目、任务和客户按工作空间归属；数据授权具备独立记录。这些保留不等同于实现 V1-B 能力。
