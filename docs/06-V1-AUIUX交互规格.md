## 1 目标、范围与体验原则

### 1.1 目标

本文件将 V1-A 产品需求、数据状态机和 API 契约转化为页面信息架构、字段、交互、异常状态、响应式与无障碍规格。它定义用户可见行为，不定义具体视觉稿、前端组件库、CSS Token 或后端实现。

### 1.2 V1-A 页面范围

| 体验域 | 页面/流程 |
|---|---|
| 账户与引导 | 注册、登录、会话失效、重认证、创建工作空间、填写画像、独立授权。 |
| 个人经营 | 工作台、项目列表/详情、三级任务、客户列表/详情。 |
| AI COO | 任务拆解、澄清、建议查看/编辑/确认/驳回、每日三件事。 |
| 数据权利 | 授权管理、AI 用量、导出、注销申请/宽限期/撤销。 |
| 运营 | 聚合漏斗、AI 聚合用量、受控用户配额配置。 |

不设计：多成员协作、公开个人主页/搜索、协作市场、支付、站内沟通、RAG/长期记忆、文件上传、通知中心、内容实验页面或运营端私密业务浏览。

### 1.3 体验原则

1. **先经营，后 AI**：没有 AI 时，项目、任务、客户和每日待办仍可完整使用；AI 入口永远有手动替代路径。
2. **AI 先解释，后确认**：AI 建议必须展示依据、边界、可编辑内容与确认后的影响；不以默认勾选、自动跳转或隐蔽按钮代替用户确认。
3. **移动优先，单手可完成**：主要操作区位于拇指可达范围；常用操作不依赖悬停；桌面端扩展信息密度而非改变任务路径。
4. **状态优先于装饰**：进度、待确认、错误、只读、冲突和撤销窗口必须明显可见，视觉装饰不得压过经营信息。
5. **渐进披露**：首次仅呈现下一步所需字段；AI 依据、审计摘要、复杂设置通过抽屉/详情展开，不阻断主旅程。
6. **可恢复性**：表单失败保留输入；并发冲突可对比并刷新；注销、导出、AI 异步均有可回查的状态入口。
7. **隐私默认**：客户备注、授权内容、AI 输入摘要默认仅本用户可见；不把敏感内容放入卡片摘要、Toast、埋点或运营页面。

## 2 信息架构、导航与响应式

### 2.1 路由与一级导航

| 路由 | 页面 | 一级入口 | 核心动作 |
|---|---|---|---|
| `/auth/login`、`/auth/register` | 认证 | 未登录入口 | 登录、注册。 |
| `/onboarding/*` | 引导 | 登录后强制流程 | 空间、画像、授权。 |
| `/dashboard` | 工作台 | 主导航“今天” | 查看三件事、快捷创建、查看待确认建议。 |
| `/projects`、`/projects/:id` | 项目 | 主导航“项目” | 创建、编辑、任务管理、AI 拆解、状态动作。 |
| `/tasks/:id` | 任务详情 | 项目/工作台深链 | 编辑、开始、完成、关闭、取消、延期。 |
| `/customers`、`/customers/:id` | 客户 | 主导航“客户” | 创建、编辑、阶段流转、查看历史。 |
| `/settings/*` | 设置 | 主导航“设置” | 画像、授权、AI 用量、导出、注销、会话。 |
| `/operations/*` | 运营 | 受控角色入口 | 查看聚合指标、配置配额。 |

### 2.2 导航结构

**移动端（<768px）**

底部固定导航为“今天 / 项目 / 客户 / 设置”；中间主操作使用页面内的显式“新建”按钮，不设置全局悬浮 AI 按钮。任务详情、项目详情、设置二级页使用顶部返回导航与清晰标题。

**桌面端（>=768px）**

左侧固定窄导航承载上述四项；右侧内容区保持单列主工作区，项目详情可在宽屏使用“主内容 + 可折叠上下文侧栏”。不得以三列等宽卡片网格替代经营列表。

**运营端**

独立 `/operations` 布局，顶部明显标注“运营聚合数据”。如果当前身份无运营权限，入口不展示；直接访问返回 403 页面，不显示私密业务替代内容。

### 2.3 响应式规则

| 场景 | 移动端 | 桌面端 |
|---|---|---|
| 列表 | 单列，卡片/行最小高度 56px，操作收在更多菜单。 | 表格化列表或两栏摘要，保留键盘可访问操作。 |
| 表单 | 全宽、单列字段；底部粘性提交区。 | 最大内容宽度 720px；关联解释可放字段右侧。 |
| AI 建议 | 全屏抽屉或独立页，固定底部“编辑 / 确认 / 驳回”。 | 右侧详情抽屉或页面内展开，确认区始终可见。 |
| 状态动作 | 使用底部 Sheet 二次确认。 | 使用紧邻按钮的确认 Popover/Modal。 |
| Toast | 顶部，单条且可读。 | 右上角，最多同时两条。 |

最小交互目标为 44x44 CSS px；正文对比度至少 4.5:1，状态不只依赖颜色表达。

## 3 全局状态、反馈与无障碍

### 3.1 统一页面状态

| 状态 | 呈现 | 用户操作 |
|---|---|---|
| 初始加载 | 内容结构 Skeleton，保留标题与导航，不使用全屏转圈。 | 可返回上一页。 |
| 列表为空 | 说明当前无数据和原因，提供一个主操作与一个合法替代路径。 | 如“尚无项目：创建项目”；任务必须在项目上下文创建。 |
| 局部提交中 | 按钮显示处理中并禁用重复点击；保留表单内容。 | 可取消仅限尚未发出请求的本地编辑。 |
| 网络/服务错误 | 中文可读说明 + 重试 + 保留输入。 | 不展示内部错误、供应商名称或堆栈。 |
| 异步处理中 | 资源状态条 + 最近更新时间；允许离开页面后回查。 | 导出/AI 均有详情入口。 |
| 并发冲突 | 不覆盖本地输入；显示“数据已更新”说明、服务端摘要和“刷新并比较”。 | 用户选择重新编辑/放弃本地修改。 |
| 只读 | 顶部持久横幅说明原因和可做动作。 | 写按钮禁用并附原因；保留查看、导出、撤销注销。 |

### 3.2 API 错误到 UI 映射

| API 错误 | UI 行为 |
|---|---|
| `400 INVALID_REQUEST` | 显示"请求格式有误，请检查输入后重试"；保留当前表单。 |
| `401 ACCESS_TOKEN_EXPIRED` | 静默尝试一次刷新；成功后重放原请求；失败跳转登录，保存当前安全返回路径。 |
| `401 UNAUTHENTICATED` / `ACCESS_TOKEN_INVALID` | 不尝试刷新，直接跳转登录；保存返回路径；显示"请登录后继续"。 |
| `403 FORBIDDEN` | 显示"你没有权限执行此操作"；不暴露具体权限规则。 |
| `403 ORIGIN_NOT_ALLOWED` / `CSRF_REQUIRED` | 显示"请求来源不被允许，请刷新页面后重试"；若持续出现提示联系支持；不暴露 Origin 白名单细节。 |
| `403 CONSENT_REQUIRED` | 在 AI 入口显示"需要授权才能使用 AI COO"，跳转至相关授权项。 |
| `403 WORKSPACE_READ_ONLY` | 显示注销宽限期只读横幅；写操作入口置灰，导出和撤销注销可用。 |
| `403 ACCOUNT_DEACTIVATED` | 显示账户已完成注销说明，不再显示业务页面。 |
| `404 RESOURCE_NOT_FOUND` | 显示"找不到该内容，可能已被取消或删除"；提供返回列表入口；不得显示"该资源属于他人"等泄露性信息。 |
| `409 RESOURCE_VERSION_CONFLICT` | 打开冲突面板；重新加载后让用户选择是否应用修改。 |
| `409 INVALID_STATE_TRANSITION` | 提示当前状态不可执行该动作，并刷新对象状态。 |
| `409 IDEMPOTENCY_KEY_REUSED` | 提示请求内容已变化，请重新发起；保留当前表单。 |
| `409 DOWNLOAD_TOKEN_CONSUMED` | 显示"下载链接已使用，请新建导出"；提供回到导出列表入口。 |
| `422 PROFILE_INCOMPLETE` | 跳转/引导到画像完成页，保留项目草稿在客户端临时态。 |
| `422 REAUTH_REQUIRED` | 打开重认证 Modal，成功后自动恢复原敏感动作的确认步骤，不自动提交。 |
| `429 AI_QUOTA_*` | 展示已达调用限制、下次可用时间或原因；提供手动项目/任务入口。 |
| `419 SESSION_REVOKED` / `REFRESH_TOKEN_REUSED` / `REFRESH_TOKEN_EXPIRED` | 清理内存令牌，显示"登录已失效，可能因你在其他设备登录或修改了密码，请重新登录"，跳转登录；不自动重试原写操作。若当前页面有未保存表单，跳转前将非敏感字段暂存到 `sessionStorage`，登录成功后恢复；密码、客户备注等敏感字段不暂存。 |
| `503 AI_CIRCUIT_OPEN` / `DEPENDENCY_UNAVAILABLE` | 显示 AI 暂不可用，保留手动拆解、手动创建任务。 |

### 3.3 交互与无障碍通用规则

1. 表单字段有可见 Label、必填标记和字段下方错误信息；不能仅用 placeholder 表达字段语义。
2. Modal/Sheet 打开时焦点移入，关闭时回到触发控件；支持 Esc（移动端保留关闭按钮）。
3. 异步状态、AI 事件、错误与成功提示使用 `aria-live="polite"`；严重错误可使用 `assertive`，但不得频繁打断。
4. 所有图标按钮包含可见文本或 `aria-label`；状态标签同时含文字，如“已暂停”“处理中”。
5. 动效只使用 transform/opacity，时长 150–300ms；遵守 `prefers-reduced-motion`，不使用自动闪烁或连续装饰性动画。
6. 删除/取消/注销等不可逆或高影响操作必须有二次确认、可理解的后果说明与取消出口。

## 4 账户、引导与设置

### 4.1 注册与登录

**注册页字段**

| 字段 | 规则 | 交互 |
|---|---|---|
| 手机或邮箱 | 至少一个；即时格式校验。 | 用户选择一种方式，另一种可选。 |
| 密码 | 密码策略说明在输入框下方；不显示具体哈希/安全实现。 | 支持显示/隐藏；提交前显示弱密码提示。 |
| 必要服务说明 | 展示服务条款/隐私政策版本并要求用户明确接受；不与可选授权混为一个勾选框。 | 注册审计记录版本；创建工作空间时创建 workspace 级 `CORE_SERVICE=GRANTED`，首次设置页可查看。 |

登录失败时不说明账号是否存在；提示“账号或密码不正确，请重试”。触发单维度限流时显示剩余等待时间；账号整体冻结时显示验证解锁入口，不能暴露 IP 命中细节。

### 4.2 引导流程

```text
注册/登录成功
  → 创建工作空间
  → 填写基础画像
  → 查看并设置可选授权
  → 进入工作台
```

| 步骤 | 内容 | 必填/可跳过 | 完成条件 |
|---|---|---|---|
| 创建工作空间 | 名称、简介（可选） | 名称必填 | 创建成功，进入画像。 |
| 基础画像 | 技能（最多 20 项）、创业阶段、经营目标 | 三项均必填 | `profile.completedAt` 有值，才可创建项目。 |
| 可选授权 | AI 业务数据、模型改进、个性化、营销 | 均可跳过 | 显示“稍后在设置中调整”；跳过不阻断工作台。 |

引导页顶部展示 1/3、2/3、3/3 进度，但不允许跳过工作空间/画像必填步骤。引导恢复基于**服务端状态**而非前端草稿：用户重新登录后，服务端返回 `workspace` 是否存在与 `profile.completedAt` 是否有值，前端据此跳转到首个未完成步骤；已完成的步骤不重复展示。

### 4.3 设置页

设置首页使用分组列表：账户安全、身份与画像、数据授权、AI 使用量、数据导出、注销账户。

| 子页 | 内容与操作 |
|---|---|
| 账户安全 | 当前会话列表、撤销单个会话、修改密码。修改密码和注销均先触发重认证。 |
| 身份与画像 | 修改空间名称/简介、画像字段、私密可见性说明。工作空间改名次数/年度剩余次数可见。 |
| 数据授权 | 每项展示用途、当前状态、政策/目的版本、授予/撤回时间、受影响能力；`CORE_SERVICE` 仅说明不可撤回。 |
| AI 使用量 | 今日/月度调用数、能力类型配额、预估费用、限制说明；不展示供应商密钥。 |
| 数据导出 | 见 §8.1。 |
| 注销账户 | 见 §8.2。 |

### 4.4 重认证 Modal

触发来源：修改密码、撤回敏感授权、提交注销、撤销注销。Modal 显示“为了保护你的账户，请再次验证身份”，仅输入密码；成功后仅更新**当前 Session**的 15 分钟重认证窗口，关闭 Modal 并回到原动作的确认步骤，不自动提交高影响操作。其他设备/Session 不共享该窗口。失败保留 Modal 和安全错误提示；关闭则放弃该动作。

## 5 工作台、项目与任务

### 5.1 工作台 `/dashboard`

页面按优先级自上而下：

1. **今日三件事**：最多三项任务及排序理由；无 AI 建议时显示活跃任务降级列表和“生成今日建议”按钮（如已授权）/“创建任务”按钮。
2. **待确认 AI 建议**：数量徽标与最近三条摘要；点击进入建议详情。
3. **活跃项目**：最多三个项目，展示状态、任务完成数、延期标记和更新时间。
4. **快捷创建**：新建项目、录入客户；已有项目时可先选择项目再手动新建任务。移动端使用展开式操作菜单，桌面端用主按钮组。

`POST /dashboard/daily-top3/actions/viewed` 在用户实际看到模块 50% 以上且停留 1 秒后上报；不因页面预加载触发。

### 5.2 项目列表 `/projects`

默认按最近更新倒序，提供状态筛选：草稿、进行中、已暂停、已完成、已取消。每行展示项目名称、状态、目标摘要（最多两行）、任务进度、计划结束日、最近更新时间。

**空状态**：

```text
还没有项目
从一个正在推进的目标开始。你可以自己建立，也可以稍后让 AI COO 协助拆解。
[创建项目]
```

### 5.3 创建/编辑项目

| 字段 | 创建 | 编辑 | 校验/说明 |
|---|---|---|---|
| 项目名称 | 必填 | 可编辑 | 最多 160 字。 |
| 项目目标 | 必填 | 可编辑 | 最多 4000 字；AI 仅使用处理后的必要摘要。 |
| 交付物 | 可选 | 可编辑 | 最多 2000 字。 |
| 计划开始/结束日 | 可选 | 可编辑 | 结束日不得早于开始日。 |

创建成功进入项目详情，初始状态为 `DRAFT`。表单草稿仅存在当前浏览器内存；刷新/关闭前提示“尚有未保存修改”，但 V1-A 不实现跨设备草稿同步。

### 5.4 项目详情 `/projects/:id`

页面结构：

| 区域 | 信息 | 操作 |
|---|---|---|
| 顶部摘要 | 名称、状态、目标、计划时间、任务进度。 | 编辑（仅 `DRAFT/IN_PROGRESS/PAUSED` 可编辑；`COMPLETED/CANCELLED` 只读）、更多状态动作。 |
| 任务区 | 三级任务树、完成数、延期/逾期标记。 | 新建任务、展开层级、进入任务详情。 |
| AI COO 区 | 未授权/可生成/处理中/建议待确认/失败降级态。 | “请 AI 拆解任务”或“手动添加任务”。 |
| 最近活动 | 当前项目的安全摘要（如状态变化时间），不展示完整审计日志。 | 无直接编辑。 |

**项目状态动作**

| 当前状态 | UI 可见动作 | 确认说明 |
|---|---|---|
| `DRAFT` | 开始、取消 | 开始前显示“至少需要一项已确认或进行中任务”。 |
| `IN_PROGRESS` | 暂停、标记完成、取消 | 完成前检查未终态任务；取消前要求处理子任务。 |
| `PAUSED` | 恢复、标记完成、取消 | 恢复后任务仍保留原状态。 |
| `COMPLETED` / `CANCELLED` | 查看 | 不显示编辑/状态动作。 |

“标记完成”不是 AI 自动动作。若所有非取消任务已结束，显示非阻断提示“所有任务已处理完，可以标记项目完成”，用户仍需点击确认。

### 5.5 任务树与任务详情

**任务树规则**

- 根任务、子任务、孙任务以缩进和层级标签显示，最大三级；新建子任务入口只出现在深度 <3 的任务旁。
- 每行展示标题、状态、截止日期、延期/逾期标记和来源（“手动”或“来自 AI 建议”）。
- 勾选完成不能跨越状态：`CONFIRMED` 先显示“开始执行”，`IN_PROGRESS` 才可完成。

**任务详情字段**

| 字段 | 编辑规则 |
|---|---|
| 标题、描述、截止时间、预估工时 | `DRAFT/CONFIRMED/IN_PROGRESS` 可编辑；终态只读。 |
| 负责人 | 只显示当前空间所有者，不提供选择器。 |
| 父任务 | 仅创建时选择；变更层级需通过受控编辑，不能超过三级。 |
| 来源 | 只读；AI 来源可链接到建议详情。 |

**任务动作与确认**

| 当前状态 | 操作 | UI 行为 |
|---|---|---|
| `DRAFT` | 确认 | 补全必填字段后变为已确认。 |
| `CONFIRMED` | 开始 | 单次确认后进入进行中。 |
| `IN_PROGRESS` | 完成、标记延期、取消 | 延期为开关/说明，不改变主状态；完成后可关闭。 |
| `COMPLETED` | 关闭 | 二次确认说明“关闭后不可编辑”。 |
| `DRAFT/CONFIRMED/IN_PROGRESS` | 取消 | 有非终态子任务时显示处理选项：先处理 / 一并取消。 |

## 6 AI COO 交互规格

### 6.1 AI 入口与授权前置

AI 入口只出现在：项目详情的“请 AI 拆解任务”、工作台的“生成今日建议”。

| 前置状态 | UI 呈现 |
|---|---|
| 未完成画像 | 入口禁用，说明“先完成画像，才能让建议更贴合你的经营目标”，跳转画像。 |
| `AI_BUSINESS_DATA` 未授权 | 入口可见但不直接调用，打开授权说明 Sheet，含“查看数据授权”“暂不使用 AI”。 |
| 工作空间只读 | 入口禁用，说明注销宽限期内无法发起新的 AI 调用。 |
| 配额耗尽 | 入口禁用或打开说明，展示下次可用时间（若 API 提供），保留手动替代。 |
| AI 可用 | 打开生成确认 Sheet。 |

生成确认 Sheet 应解释：AI 将读取已授权的当前空间结构化项目/任务/客户摘要；客户名称会以别名处理；不会自动创建或修改任务；用户可在生成后编辑、确认或驳回。

### 6.2 AI 生成与 SSE 过程

用户确认生成后，调用 `POST /ai/runs` 并进入页面内处理态：

| SSE 事件 | UI 状态 | 可用操作 |
|---|---|---|
| `run.created` | “正在准备你的经营信息” | 返回项目/工作台；run 继续后台执行。 |
| `run.progress` | 阶段文案：“正在整理项目目标”“正在生成可执行建议”。 | 取消仅关闭界面，不取消已持久化 run。 |
| `clarification.required` | 澄清问题卡片。 | 填写补充信息后创建新 run；不继续猜测。 |
| `suggestion.ready` | 跳转/展开建议详情。 | 编辑、确认、驳回。 |
| `run.degraded` | “未能生成结构化任务，已给出参考建议”。 | 复制/参考、手动创建任务；不显示一键写入。 |
| `run.failed` | “AI COO 暂时不可用”。 | 重试（产生新 run，如配额允许）、手动管理。 |
| 断线 | “连接已断开，建议仍在处理中”。 | 自动用 `GET /ai/runs/{id}` 轮询恢复，5 秒间隔为 UX 建议值（可在 07 性能测试中调整），总轮询不超过 run 创建后 60 秒；之后提示“稍后在建议列表中查看”。 |

禁止显示模型逐 token 原文、系统 Prompt、供应商名称、未脱敏客户信息或内部失败细节。首个反馈在 3 秒内出现；15 秒未终态时改为“仍在处理中，可稍后查看”，不阻塞用户继续手动管理。

### 6.3 建议详情与确认

建议详情显示：建议类型、生成时间、AI 标识、项目/任务关联、排序/拆解理由、模型版本摘要、已使用数据类别与授权依据摘要、`traceId`（可复制但不作为面向普通用户的调试工具）。

**任务拆解建议**

- 使用可编辑任务列表，每项含标题、描述、截止时间（或“待确认”）、预估工时、依赖关系、层级预览；前端内部保留 API 返回的稳定 `itemKey`，不展示为可编辑字段。
- 编辑仅改动即将确认的 `editedPayload`，原始建议保持可查看；编辑后的字段有“已修改”标记。
- 明确提示“确认后将创建 N 项已确认任务；不会自动开始或关闭项目”。
- 确认按钮为“确认并创建任务”，驳回按钮为“这次不采用”。两者同级但确认是主操作。

**每日三件事建议**

- 显示最多 3 项关联任务、排序和理由；点击任务进入任务详情。
- 用户可调整排序或移除一项；确认时提交 1–3 项有序 `items=[{taskId, rank}]`，省略某项表示移除；确认按钮为“确认今日重点”，不会创建/修改任务。
- 未确认也不妨碍用户直接变更任务状态。

**确认/驳回结果**

| 动作 | 成功 UI | 失败/冲突 UI |
|---|---|---|
| 确认任务计划 | 显示创建任务数量，跳转项目任务区并高亮新任务。 | 保留编辑内容；版本冲突显示刷新比较；执行失败提示手动创建或新开 run。 |
| 驳回 | 建议状态改为“已驳回”，从待确认列表移除。 | 保留当前页，可重试请求；不创建任务。 |
| 确认三件事 | 显示“今日重点已更新”。 | 保留排序选择并重试。 |

**非待确认状态的建议详情**

| 建议状态 | 详情页呈现 | 可用操作 |
|---|---|---|
| `EXECUTION_FAILED` | 标记“确认失败”；显示失败原因摘要（稳定错误码对应中文），不显示确认/驳回按钮。 | “手动创建任务”（跳转任务新建页并预填建议内容）、“重新生成建议”（创建新 run，如配额允许）；不提供“重新确认”。 |
| `EXPIRED` | 标记“建议已过期”；不显示确认/驳回按钮。 | “重新生成建议”入口。 |
| `REJECTED` | 标记“已驳回”；显示驳回原因（如有）。 | 仅查看，不提供重新确认；可新建 run。 |
| `CONFIRMED` | 标记“已确认”；显示已创建的资源链接。 | 跳转到已创建的任务/项目。 |

### 6.4 AI 透明与反馈

建议详情底部提供“建议有误？”入口，V1-A 打开轻量反馈 Sheet：错误类型（事实不符/不可执行/重复/其他）和可选说明。该反馈仅作为前端产品事件与后续质量样本输入，**不承诺自动更正建议或训练模型**。提交后显示“反馈已记录，你仍可自行调整任务”。

## 7 客户与 CRM-lite

### 7.1 客户列表与详情

客户列表按最近更新排序，支持阶段、意向度筛选。每行展示名称、阶段、意向度、下一步动作、更新时间；不展示完整备注。

| 页面 | 主要内容 | 主要操作 |
|---|---|---|
| 客户列表 | 阶段筛选、客户行、空状态。 | 新建客户、进入详情。 |
| 新建客户 | 名称、来源、意向度、下一步动作、备注。 | 保存；初始阶段为 `LEAD`。 |
| 客户详情 | 基础信息、备注、当前阶段、下一步动作、阶段历史。 | 编辑、改变阶段。 |

备注区域注明“仅你可见；不会默认发送给 AI COO”。

### 7.2 客户阶段变更

阶段操作不是自由下拉编辑。点击“改变阶段”打开 Sheet，展示当前阶段、允许的目标阶段、可选原因输入和变更后的含义。

| 当前阶段 | UI 允许的下一阶段 | API 枚举映射 |
|---|---|---|
| `LEAD`（线索） | 已联系、已失去 | → `CONTACTED`、`LOST` |
| `CONTACTED`（已联系） | 已验证、已失去、回到线索 | → `QUALIFIED`、`LOST`、`LEAD` |
| `QUALIFIED`（已验证） | 已赢得、已失去、回到已联系 | → `WON`、`LOST`、`CONTACTED` |
| `WON`（已赢得） | 无后续变更；说明 V1-A 只记录 CRM 结果，不代表合作/支付。 | 终态 |
| `LOST`（已失去） | 重新作为线索、重新联系 | → `LEAD`、`CONTACTED` |

成功后在详情顶部更新阶段，并在历史区追加“操作人、时间、前后阶段、原因（如有）”。阶段变更冲突时刷新当前阶段后让用户重新选择。

## 8 数据权利与生命周期

### 8.1 数据导出

导出页显示范围说明：“将导出当前工作空间的项目、任务和客户基础数据，格式为 CSV；不包含密码、会话令牌、原始 AI Prompt/输出。”

| 状态 | UI 呈现 | 可用操作 |
|---|---|---|
| 无导出记录 | 主按钮“创建数据导出”。 | 创建。 |
| `REQUESTED/QUEUED/GENERATING` | 处理中状态、创建时间、可离开说明。 | 刷新状态；不可重复领下载令牌。 |
| `READY` | 完成时间、到期时间、一次性下载说明。 | “获取一次性下载链接”。 |
| `DOWNLOADED` | 已下载时间。 | 不再提供同一令牌；可新建导出。 |
| `FAILED` | 安全失败说明和重试按钮。 | 新建导出（新幂等键）。 |
| `EXPIRED` | 已过期说明。 | 新建导出。 |

下载流程：用户点“获取一次性下载链接”后取得令牌；前端立即以 POST body 发起下载，成功后浏览器开始下载/跳转至短效 URL。页面不得显示、复制或在地址栏暴露一次性 token。下载失败但 token 未消费时，说明可重新尝试；已消费时显示“链接已使用，请新建导出”。

### 8.2 注销申请、宽限期与撤销

**申请注销流程**

```text
设置 > 注销账户
  → 后果说明
  → 重认证
  → 最终确认
  → 宽限期只读页
```

后果说明必须明确：

1. 提交后项目、任务、客户和 AI COO 将不可新建或修改。
2. 用户在 30 天内仍可登录查看历史数据、导出基础数据并撤销注销。
3. 宽限期结束后系统匿名化主库数据，不能恢复；存在法定/争议保留义务时会单独显示范围和期限。

最终确认使用明确动作词“确认申请注销”，不使用模糊“确定”。提交成功后跳转注销状态页，显示宽限结束日、当前状态、导出入口和“撤销注销申请”按钮。

**宽限期全局体验**

- 顶部固定横幅：“账户正在注销宽限期。你可以查看、导出或撤销申请；不能新建或修改经营数据。”
- 所有写按钮禁用，并将原因关联至横幅；AI 入口显示同样原因。
- 允许调用只读列表/详情、导出、注销状态查询和撤销注销；禁止新建导出外的业务写。

**撤销注销**

点击后再次要求重认证，再显示“撤销后将恢复正常编辑和 AI COO 使用”的确认 Sheet。仅 `GRACE` 可撤销；`RETENTION_HOLD` 显示保留原因与预计结束时间、无撤销按钮；`TOMBSTONED` 展示终态说明且不允许恢复。

## 9 运营页面

### 9.1 运营首页

仅授权运营身份可访问。页面顶部声明“仅聚合指标，不含用户私密项目、任务、客户内容”。

| 区块 | 指标 | 交互 |
|---|---|---|
| 激活漏斗 | 注册、画像完成、项目创建、任务确认、每日三件事访问。 | 按日期筛选，默认最近 14 天。 |

| 阶段门 | 注册至项目、AI 确认率、7 日留存、每日三件事周活。 | 展示 PRD/SRS 固定口径与当前观测窗口。 |

### 9.2 用户 AI 配额配置

管理员通过独立 `platform_operator` 入口完成 MFA 后进入受控页面，调用 05 v1.2 的 `GET /operations/users/search?identifier=...`（手机/邮箱前缀至少 4 个字符），配置特定能力的日限额。提交前展示当前值、新值和影响：“该设置只影响 AI 调用，不影响手动经营功能。”

配额修改成功后显示审计已记录；页面不应顺带显示该用户项目、客户、任务或完整 AI 历史。

## 10 前端状态模型与埋点

### 10.1 客户端状态边界

| 状态类型 | 存放位置 | 规则 |
|---|---|---|
| 服务端事实 | API 查询缓存 | 项目、任务、客户、建议、授权、导出、注销状态均以 API 响应为准。 |
| 表单草稿 | 页面局部状态 | 未保存项目/任务/客户表单仅当前会话内保存；不作为业务事实。 |
| Access Token | 内存 | 刷新页面后通过 Refresh Cookie 重新获取；禁止持久化到 LocalStorage。 |
| UI 偏好 | 本地存储可选 | 仅导航收起、筛选等非敏感偏好；不得存 token、客户备注、AI 原始内容。 |
| SSE 连接 | 页面局部状态 | 断线后基于 `runId` 查询恢复；不缓存原始模型流。 |

### 10.2 页面级埋点

所有埋点遵循 PRD §8 公共字段；不得包含密码、令牌、原始证件、客户完整备注、原始 Prompt/输出。

| 事件名 | 触发时机 | 额外字段 |
|---|---|---|
| `workspace_created` | 工作空间创建成功 | — |
| `profile_completed` | 完整画像首次提交成功 | `stage_label` |
| `project_created` | 项目创建成功 | `source=manual/ai_confirmed` |
| `task_status_changed` | 任务状态动作成功 | `before_status`、`after_status`、`source` |
| `ai_task_generation_requested` | AI run 创建成功 | `project_id`（仅内部标识） |
| `ai_task_generation_completed` | run 进入 `GENERATED/DEGRADED/FAILED` | `duration_ms`、`result_type`（值域：`generated`/`degraded`/`failed`/`timed_out`） |
| `ai_suggestion_viewed` | 建议详情实际可见 | `suggestion_type` |
| `ai_suggestion_confirmed` | 建议确认事务成功 | `edited_count` |
| `daily_top3_viewed` | 工作台三件事模块实际可见 | `source=ai/fallback` |
| `customer_created` | 客户创建成功 | `source` |
| `consent_changed` | 授权变更成功 | `consent_type`、`before`、`after` |
| `data_export_requested` | 导出任务创建成功 | — |
| `deactivation_requested` | 注销进入宽限期成功 | — |

### 10.3 UI 状态与 API 状态一致性

前端不得在服务端确认前乐观标记“已完成”“已确认”“已下载”“已注销”。可展示按钮处理中态，但业务状态只能在成功响应或 SSE 终态后更新；版本冲突、断线和失败均须重新拉取服务端事实。

## 11 页面验收与下游交接

### 11.1 UI/UX 最小验收场景

1. 新用户可在 3 分钟内完成注册、工作空间和画像；未完成画像不可创建项目。
2. 用户可不使用 AI 创建项目、三级任务和客户，并完成合法状态转换。
3. AI 建议在确认前不出现在任务树；编辑后确认才创建 `CONFIRMED` 任务。
4. AI 断线、超时、Schema 降级、未授权、配额耗尽时均提供手动替代路径。
5. 跨空间资源、版本冲突、会话失效、重认证和只读状态均有明确且不泄露信息的 UI。
6. 导出在处理中/完成/失败/过期均可回查；一次性令牌不进入地址栏或页面文本。
7. 注销宽限期保留查看、导出、撤销，禁用业务写与 AI；tombstone 后不显示可恢复入口。
8. 移动端所有主要操作可键盘/读屏访问，触控目标满足最小尺寸，减少动效模式可用。

### 11.2 07 测试与发布计划交接

07 必须覆盖：移动/桌面断点、键盘/读屏、表单保留、所有 API 错误映射、SSE 断线恢复、并发冲突、状态动作确认、AI 数据最小化提示、导出 token 不泄露、注销宽限期只读和运营聚合权限边界。
